推特Article大纲：Autoresearch — AI做量化研究的完整实验记录
叙事主线：一个想法如何通过30+次实验迭代，从"预测操纵"的幻想走到"顺势做空"的实战

开头 Hook
我让AI agent跑了30+轮实验，试图破解妖币庄家操纵的密码。最后的结论和最初的假设完全相反。

这不是成功案例的PR稿。这是一份完整的失败和修正记录。

Part 1: 起点 — 一个看起来很合理的假设
假设：妖币操纵前一定有信号（FR异常、OI堆积、链上异动）。找到信号 → 提前布局 → 印钱。

数据准备：16个币种，82个人工标注事件，12个数据维度（FR/OI/爆仓/多空比/链上/订单簿/taker）

数据点：

16 symbols, 82 labeled events
12+ data dimensions (on-chain, orderbook, funding, OI, liquidation, taker flow)
Part 2: Phase 1 — 信号检测的12轮实验
叙事：从0.19到0.72再到0
按因果链讲述：

FR baseline: F1=0.19, 只靠funding rate，recall 18%
+爆仓量: F1跳到0.47 — 爆仓量是核心信号，不是FR
+链上数据: F1=0.39 — 链上数据没用，跟volume完全共线
+分类系统: F1=0.34 — 分5类反而更差
换ML(XGBoost): F1一步跳到0.60 — ML暴力提升
ML调优: F1→0.67→0.72 — precision=recall=72%
OOS测试: F1=0, 25个事件0命中 — 完美过拟合
关键数据点：

Train F1: 0.19 → 0.72 (3.8x improvement)
OOS F1: 0.000 (zero, not rounding error)
On-chain data AUC vs volume: all < 0.55 (random)
Insight #1: "预测操纵什么时候发生"从数据角度不可解。操纵前是平静期——所有活跃度指标都低于baseline。庄家等市场安静了才动手。

Part 3: 思路转向 — 从预测到反应
转折点：接受"预测不可能"，转向"检测操纵正在发生 → 顺势交易"

新数据：415个完整操纵周期（vs之前的82个碎片事件），22 → 55币种

推翻了什么：

操纵前活跃度高 → 操纵前是平静期（旧结论因事件定义碎片化导致误判）
差庄指纹确认：pump期间flash candle频率 vs dump深度 r=0.40***
数据点：

415 → 1447 manipulation events across 55 symbols
Pre-manipulation = calm period (all activity metrics BELOW baseline, p<0.001)

Part 5: 做空最终版 — 数据说话
为什么只用K线？（排除法全链路）
最终策略只用K线结构，没有任何量化指标。不是没考虑——是12个维度、63个特征全测过后的排除法结论。

三个阶段的排除过程：

Phase 1信号检测（12轮实验）: FR/链上/OI/多空比/订单簿 → 全部与volume共线或无增量。ML组合全部特征 → Train F1=0.72, OOS=0（完美过拟合）

Phase 2做空入场filter: flash candle(加了更差)/vol_ratio(AUC 0.537)/taker ratio(AUC 0.532) → 全部接近随机。Peak detection模型AUC=0.952但不能区分操纵vs正常pump。唯一强信号pump_pct(AUC=0.869)是事后指标。

做多策略63维度全测: 订单簿(<0.55)/CVD(数据不全)/大单(数据不全)/散户多空比(0.50完全随机)/大户多空比(<0.55)/FR(0.575弱) → K线的ret_4h(AUC=0.702)仍是最强实时信号

结论：K线结构不是"简化"，是30+轮实验排除法的终点。

数据点：

12 data dimensions tested
63 signal features evaluated
Strongest non-price signal AUC: 0.617 (OI change, weak)
Strongest price signal AUC: 0.702 (4h return)
Only strong discriminator: pump_pct AUC=0.869 — but it's a post-hoc metric

Part 5.5: 从Sharpe 15到Net归零 — 真实成本的打击
这是整个研究中最重要的转折。回测做得再好，不过真实成本这一关就是空中楼阁。

数据来源：

FR: 55币种全量历史，逐笔trade计算
Orderbook depth: Coinglass真实bids_usd/asks_usd（非模型估算）
滑点: impact = position / (2 × depth)
核心发现: 妖币orderbook depth中位数只有 $70K。

depth / 1h volume ratio = 0.020（depth只有成交量的2%）
$1K的taker order推价0.7%，来回2.34%
Raw alpha +1.06% 被滑点吃光
仓位	Slippage	Net Avg	WR	Sharpe
$200	0.57%	+0.38%	42%	0.7 (p=0.46)
$500	1.33%	-0.39%	37%	-0.7
$1K	2.34%	-1.40%	29%	-2.5
$200是盈亏平衡点，且不显著。

Sharpe的坍缩路径：

Raw: 14.3 → +FR: ~14 → +Fee: ~13 → +Slippage: ~0
不是alpha不存在，是capacity不存在。 策略确实能识别正确的做空时机（WR 90%），但妖币的流动性让执行成本>收益。

这对article是最好的结尾：回测的Sharpe 15不是假的，但它活在一个没有摩擦力的真空里。加上真实世界的一层orderbook depth，整个数字归零。量化策略的最终boss不是alpha，是capacity。

Part 7: 为什么做空有edge而做多没有
结构性原因：

做空的edge：操纵后dump是确定性的（庄家必须出货）+ support跌破是结构性信号
做多的edge：洗盘后续涨是概率性的（有些pump洗完就dump了）+ 无法实时区分操纵vs非操纵pump
做空不是在"赌它会跌"，而是在"等结构确认后顺势做空一个已经在dump的coin"。 做多是在"赌洗盘结束后会继续涨"——这个赌注没有结构性保障。

Part 7.5: V4实时化 — 从不可执行到可执行
Peak Detection：6种方案全测，2种存活
策略回测WR=71%但需要知道peak在哪。实时中不可能。测了6种前置方案：

无peak确认 → p=0.064不显著
4h上影线≥30%（V4A） → WR=60%, p=0.026 ✓
4h阴线+range≥20%（V4B） → WR=74%但Q4不稳定
大振幅bar/pump speed/4h阴线 → 全不work
关键insight：V4A更早入场（距peak 5h）但13%假信号；V4B更晚（10h）但0%假信号。两种是不同的risk-reward profile。

Exit优化：从拍脑袋到数据推导
问题：V3的trail=3%对所有策略统一使用，但V4A/V4B入场时机不同。

方法：逐bar追踪每笔trade的浮动PnL + 真实FR（coinglass 8h结算数据），找到最优出场bar的metrics。

发现：

FR成本<0.03%/trade → 不是主因
边际收益24h后变负（反弹+FR累积）
最优出场时bounce(从最低点反弹)：V4A P50=2.14%, V4B P50=2.45%
出场bar 100%是阴线 → 最优出场在"还在跌但快到底了"
"跌速减缓"信号AUC=0.521 → 无法预测最优出场
结论：前置出场信号不存在，trail是最稳健的规则。但trail值应该per-strategy：

V4A: trail=2% / V4B: trail=2.5% / V3: trail=3%
全部加8h时间兜底
真实成本检验：从"不可执行"到"limit order可执行"
初版结论（错的）：用凌晨静态orderbook snapshot（depth $9.5K），V4A扣成本后不显著。

修正：策略只在操纵breakdown时触发，此时1h成交量中位数$828万，是平时的几十到百倍。用entry bar volume估算depth = $83K-$165K，比静态snapshot高1000倍。

Market order $500: 全量+1.13% (p=0.013)，holdout +0.83% (NS)
Limit order $500-$2K: 全量+1.98% (p<0.0001)，holdout +2.14% (p=0.038)
关键发现：95%的trade在breakdown后3h内有≥0.3%反弹，limit order几乎一定能fill。

教训：成本分析必须用策略触发时刻的流动性，不能用平时的snapshot。操纵周期=高波动=高成交量=深orderbook。

Part 7.75: 过拟合测试 — 质疑自己的结论
方法论缺陷发现
之前的过拟合测试（±20% sweep, walk-forward, bootstrap）全在manip-only事件上做。但实时scanner不知道pump是否会dump。721事件研究集只选了41%会dump的，另外59%被预先筛掉了——这就是选择性偏差。

需要四层验证：币种筛选是否有效 → 策略在全量pump上是否成立 → time-series holdout → 多重比较校正。

Level 1: Base Rate的真相
70个高操纵币上pump≥20%共1890次，只有41%最终dump≥20%。对比：低操纵币15%，零操纵币0%。币种筛选有效（39% vs 15%），但不能保证每次pump都是操纵。

Level 2: Time-series Holdout — V4B崩了
数据按时间切70/30，前70%开发，后30%盲测：

策略	Dev Avg	Holdout Avg	Holdout p
V4A	+1.95%	+2.15%	0.033
V4B	+4.71%	+0.30%	0.84
V3	+1.76%	-0.17%	0.81
V4B过拟合。V3不行。V4A是唯一在holdout上存活的策略。

V4A holdout 28笔细看：manip事件WR=71%/+3.31%，non-manip WR=27%/+0.37%。亏损很小因为trail兜住了。

Level 3: 全量pump验证
在1890个pump≥20%事件上（不筛dump），V4A仍然N=100, WR=61%, +2.01%, p<0.001。

Score越高表现越好（全市场+1.23% → score≥1.0 +1.85%，单调递增）。在低操纵币上V4A不显著（p=0.24）——该不work的地方不work。

操纵特征指纹 — 事件级vs币种级
Manip vs Non-manip区分力最强的特征：max_range(AUC=0.847)、pump_speed(0.765)、pump_pct(0.730)。pump≥50%的base rate从22%跳到69%。

但这些特征在事件级有效，在币种级无用——一个高操纵币既有猛pump也有温和pump，平均值把信号稀释了。测了8种改进score方案，holdout base rate全在25-28%，无差别。

Event filter（pump≥50%等）在holdout上也没改善策略——filter后trade数从28降到7-12，统计力不足。且被砍掉的non-manip trade平均只亏0.37%，不值得。

过拟合的诚实结论
V4A是真的 — holdout正、全量pump正、全市场正、score单调递增
V4B过拟合了 — dev漂亮，holdout崩
V3不行 — holdout负
Event filter先不加 — trade数太少，等forward test积累数据
Part 7.85: V5扩池 — 用数据验证"老币假说"
起因
V4A过拟合测试做完，holdout只有30笔，统计力不够。Decay score在dev上r=+0.41强但holdout p=0.086不显著。不确定是过拟合还是样本量问题。

假说
领导观察：2023年上线的老币，到了2025年合约庄家玩法成熟期，反而是更纯粹的操纵场景：

项目方筹码已释放/OTC转卖完，流通盘小且集中
已有Binance合约listing，庄家可以直接用合约工具
没有持续的token unlock解锁压力
没有项目方拉盘，pump必然是"人为推动"
如果V4A在老币上也work（甚至更强），就是反过拟合的强证明。

V5池子重构
扩展到本地全部536 symbols，按最近6个月operating score重新分类：

超高操纵 75 / 高 62 / 中 23 / 低 376
新研究池 = 超高+高 = 137个（V4老池70 → V5新池137）
V4A backtest扩到166笔
全量: WR=62%, Avg=+1.73%, p<0.0001
Holdout 50笔: WR=50%, Avg=+1.34%, p=0.034 ✓
决定性验证：按era切分
Era	N	Avg	p	Manip-only Avg
Mid (2023-09到2025-02)	31	+2.76%	0.009	+5.33%
New (2025-03+)	132	+1.50%	<0.0001	+2.82%
Mid era的manip-only平均赚5.33%，是新币的1.9倍。 老币假说成立。

Decay Score在不同era的有效性
Era	r (Spearman)	p
Mid era	+0.619	0.0002
New era	+0.206	0.018
Mid era上decay score r=0.619，是new era的3倍。

关键结论
V4A是真实alpha — 跨era都work，且在更纯粹的mid era上效应更强
Decay score不是small-sample artifact — 在两个独立era上都显著
New era有污染 — 项目方运营/解锁/基本面噪音都在污染pattern
未来研究集应该向mid era倾斜 — 当前pool里mid era只占19%但贡献最大alpha
Part 8: 方法论总结
什么work了
从结果倒推 > 先假设再验证
参数全数据推导 — 每个数字有来源（MAE分位数/bounce P50/FR结算频率）

质疑自己的过拟合测试 — 参数robust≠策略robust，需要检验整个开发过程
什么被推翻了
操纵可以预测 → 操纵前是平静期
链上数据有用 → 全部与volume共线
ML能解决 → Train F1=0.72, OOS=0
Peak附近能交易 → MAE太大
做多做空对称 → 做空确定性 vs 做多概率性
越精确瞄准操纵越好 → 越瞄准越亏
回测Sharpe=实际Sharpe → 滑点让Sharpe 15归零
Peak可以精确检测 → 所有前置方案AUC<0.58
On-chain指标能区分真假breakdown → 全部AUC<0.58
最优出场点可以预测 → 跌速减缓AUC=0.52，trail是唯一可行方案
统一exit参数 → V4A/V4B入场不同，最优trail不同

改进操纵score能提升表现 → 事件级特征有效，币种级无用
新币上线越早越乱 → 反过来：老币（项目释放完+合约工具成熟）才是最纯粹的操纵场景，V4A在mid era上alpha几乎是新币2倍
当前简单score（freq×avg_pump）有用 → 在ALL/DEV/HOLDOUT三个场景下都p>0.3，无预测力。Decay score是更优替代
结尾
从一个简单的假设（"妖币操纵前有信号"）开始，经过50+轮实验，最后得到的不是一个完美的策略，而是一系列越来越诚实的认知：

Phase 1：预测不可能（OOS=0）
Phase 2：检测可以做，但peak detection是天花板
V3：回测好看（Sharpe 15），扣完成本归零
V4：接受不精确的peak detection，换取可执行性
过拟合深度审计：V4B崩了，V4A是唯一经受住time-series holdout的策略
最终存活的是V4A — 4h上影线≥30%确认peak + 5h内support跌破3% + trail 2%。Holdout上28笔trade, WR=54%, avg=+2.15%, p=0.033。全量pump测试N=100, p<0.001。用limit order扣完成本后holdout仍然显著（+2.14%, p=0.038）。

这不是一个"我找到了alpha"的故事。这是一个"alpha可能存在但capacity极小、信号极稀疏、且只有在诚实面对过拟合后才能判断是否真实"的故事。Forward test进行中。

第六章：方法论总结
有效的方法论
所有参数从数据推导 — 每个参数有明确的数据来源（P75/P90/AUC等）
信号层大TF + 执行层小TF — 1h识别结构 + 1m精确入场

三项防过拟合检验 — 参数±20% + cross-symbol + walk-forward split
做空用trail exit而非固定TP — 顺着走势走，不预设目标
被推翻的假设
操纵可以预测 → 操纵前是平静期，无预测信号
链上数据有独立信息 → 全部与volume共线
ML能解决信号检测 → OOS = 0，完美过拟合
Peak detection能交易 → 60%假阳性，MAE太大
越瞄准操纵越赚 → 越瞄准操纵越亏
做多做空对称 → 做空有结构性edge，做多没有
洗盘后必续涨 → 仅在操纵context下概率性成立，不能独立交易
核心insight
做空的edge来源：操纵后dump是确定性的（庄家必须出货），support跌破是结构性信号
做多没有对称edge：洗盘是概率性的（有些pump洗完就dump了），且无法实时区分操纵vs非操纵pump
